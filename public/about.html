About AI Guard Manager

Mission

To protect AI usage across the organization through transparent governance of models, personas, agents, and guardrails - with support for auditability, testing, and real-time enforcement.

Core Components

  • Agent: The complete deployed AI assistant
  • Persona: Defines tone, identity, and language behavior
  • Guardrail: Enforces compliance, ethics, and safety policies
  • Model: The LLM (e.g., GPT-4) that powers responses

Key Features

  • Persona Builder: YAML editor with versioning and response examples
  • Guardrail Manager: Block/log/notify rules with compliance mappings
  • Agent Rule Builder: Assemble persona + guardrails + model for a goal
  • Model Config: Set temperature, token limits, system prompt, etc.
  • Audit Logging: Full change history, file-level audit, test tracking
  • SSO Support: OpenID Connect with auto user provisioning and roles

Workflow Diagram

File Structure

/data/
  +-- persona/         # AI personalities
  +-- guardrails/      # Policy enforcement rules
  +-- agent_rules/     # Full agent YAML configurations
  +-- models/          # Model configurations

Admin & Tools

  • admin_console.php: Audit logs, test runner, system health
  • user_admin.php: SSO roles, access levels, activity
  • run_test.php: YAML test cases with real LLM inference
  • install_checker.php: Validates environment config & extensions

Roadmap

  • Nightly AI response drift tests (Almost complete)
  • Real-time guardrail enforcement (planning)
  • LangChain/LLM orchestration integration (planning)
  • DSAR & GDPR traceability reports (planning)
