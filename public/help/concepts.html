AI Guard Manager: Key Concepts

1. Agent

Definition: A complete AI-based entity that performs tasks autonomously or semi-autonomously.

Characteristics:

  • Has a defined purpose or goal
  • Includes a Model, Persona, and Guardrails
  • Can interact with users or systems

Example:

TravelAdvisorBot = Model + TravelPersona + Guardrails

2. Persona

Definition: The AI’s personality, tone, and communication style.

Characteristics:

  • Defines tone, role, and identity
  • Helps shape consistent AI output

Example:

name: Travel Assistant
tone: Friendly and energetic
role: Recommends vacations without making bookings

3. Guardrail

Definition: Security, compliance, or risk control rules that restrict AI behavior.

Characteristics:

  • Block or log unsafe input/output
  • Used for ethics, legal, and policy enforcement

Example:

aml_guardrail.yaml:
- Block money laundering prompts
- Log responses about financial crimes
- Notify admin if triggered

4. Model

Definition: The large language model (LLM) that generates the responses.

Characteristics:

  • Processes prompts and returns completions
  • Examples: GPT-4, Claude, LLaMA, Gemini

Example:

gpt-4:
- Temperature: 0.3
- Max Tokens: 2048
- System Prompt: "You are a travel expert..."
