Web Application Firewall (WAF) Help

The WAF feature in the AI Management Portal helps protect your application by filtering malicious requests and blocking potentially harmful access patterns.

WAF Settings Explained

  • Enable WAF: Master switch to turn protection on or off.
  • SQL Injection Protection: Blocks known SQL injection patterns in user input.
  • XSS Protection: Blocks attempts to inject malicious scripts via forms or URLs.

IP and Country Restrictions

You can control access based on IP addresses and geographic location:

  • IP Mode: Choose between:
    • Block All Except - Only listed IPs can access.
    • Allow All Except - Everyone can access except listed IPs.
  • Country Mode: Similar to IP mode, this uses geolocation to block or allow countries.
  • Format: Use ISO country codes (e.g. US, DE) separated by commas.

Rate Limiting

Prevent abuse by limiting repeated failed access attempts:

  • Enabled: Blocks any IP after 10 denials within 10 minutes.
  • Auto-unblocks: After 10 minutes with no new blocks.

JSON-Compatible API Blocking

Enable this if your portal is used by API clients. When triggered:

  • Requests with Accept: application/json return a structured JSON error.
  • Prevents HTML errors in API responses.

WAF Denial Log

The denial log shows the last 50 blocked attempts. Logged details include:

  • IP address
  • Country code
  • Reason for block
  • URL path and user-agent

Best Practices

  • Regularly audit your allowed IPs and countries.
  • Monitor the denial log for suspicious patterns.
  • Combine WAF with secure coding and authentication practices.
For questions or feedback about WAF, contact your system administrator or open a support ticket via the admin console.
